# Agent Instructions

## UI Design

- For all future UI design and frontend implementation work, read and follow `ref/ui/airtable/DESIGN.md` before making changes.
- Treat `ref/ui/airtable/DESIGN.md` as the local design reference for color, typography, spacing, radius, and component tone.
- Keep the desktop app sober, workflow-focused, and dense enough for repeated use. Prefer white canvas, dark ink, restrained borders, modest typography, and near-black primary actions.
- Do not introduce unrelated visual systems, decorative gradients, or generic AI-style styling when working on UI.

## Packaging

- Before creating any local package or release package, bump the relevant package version so installers can upgrade over the previous build.
